# Iconography

QORWAY iconography must be geometric, restrained, and system-oriented.

## Principles

- Stroke-based icons
- Geometric construction
- No gradients
- No playful metaphors
- No cartoon AI symbols
- Prefer shield, cube, graph, node, hexagon, route, lock, document, trace

## Runtime icon mapping

| Runtime layer | Icon direction |
|---|---|
| Data | database / grid |
| Graph | node network |
| Atlas | eye / neural grid / compass |
| Domain Packs | cube / modular block |
| PolicyCore | shield / lock / rule |
| PulseFlow | route / pulse line |
| GreenCore | leaf / energy node |
| Execution | play / terminal |
| Feedback | loop / return arrow |
| Audit | document / trace / stamp |

## Motifs

### Dot-matrix

Used for:

- European data cartography
- Atlas visual fields
- complex graph surfaces
- sovereign infrastructure maps

### Hexagons

Used for:

- modular architecture
- Domain Packs
- graph entities
- system nodes

### Shields

Used for:

- sovereignty
- tenant boundary
- PolicyCore
- security
- audit validation
