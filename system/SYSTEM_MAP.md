# System Map

**QORWAY Decision Intelligence Infrastructure**

> From fragmented enterprise systems → to causal, constrained, governed, and resilient decision infrastructure.

---

## 1. Definition

The QORWAY System Map defines how the full infrastructure operates as a unified decision intelligence runtime.

It shows how data becomes reasoning, reasoning becomes governed decisions, decisions become execution, and execution becomes feedback.

QORWAY is not a static architecture.

It is a closed-loop system.

## 6. Supporting System Layers

### Knowledge Graph Layer

The Knowledge Graph is the memory and causal structure layer.

It stores public-safe categories such as entities, indicators, actions, risks, dependencies, constraints, decisions, events, outcomes, and feedback.

It supports Universal Knowledge Graph and Private Tenant Knowledge Graph concepts.

### Agent Runtime Layer

The Agent Runtime executes bounded tasks generated by validated decisions.

Agents do not reason independently from Atlas.

Agents do not execute without PulseFlow.

Agents remain bounded by PolicyCore and tenant policies.

### Governance Layer

Governance is embedded across the full runtime.

It includes causal evidence logging, constraint validation, tenant boundaries, human oversight where required, execution traceability, and audit-ready logs.

### Sovereign Resilience Layer

Sovereign resilience surfaces dependency, continuity, provider, data boundary, regulatory, and autonomy risks.

It influences reasoning, validation, routing, execution, and feedback.

## 8. Decision Object Lifecycle

A QORWAY decision is not a text recommendation.

It is a structured system object.

Public-safe lifecycle:

```text
created → reasoned → contextualized → validated → approved / modified / rejected → orchestrated → optimized → executed → observed → learned
```

A decision is invalid if it cannot be traced through this lifecycle.

## 10. Separation of Concerns

| Layer | Can do | Must not do |
|---|---|---|
| Atlas | Reason causally | Execute directly |
| Domain Packs | Provide context | Override constraints |
| PolicyCore | Validate constraints | Generate decisions |
| PulseFlow | Orchestrate execution | Reason causally |
| GreenCore | Optimize execution | Override PolicyCore |
| Agents | Execute bounded tasks | Bypass PulseFlow |

## 11. Multi-Tenant Boundary

Each tenant has isolated:

- data
- Knowledge Graph context
- Domain Pack instantiation
- PolicyCore constraints
- PulseFlow execution events
- GreenCore execution policies
- audit logs
- feedback loop

Core rule:

> No tenant can access or infer another tenant's private graph or execution state.
